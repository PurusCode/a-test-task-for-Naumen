Выбранные приложения и их польза

Telegram

Польза:

  • Быстрая отправка сообщений и файлов без ограничений по размеру
  • Каналы и боты для получения новостей и автоматизации задач
  • Синхронизация между устройствами с сохранением истории переписок

Частота использования: Ежедневно

Яндекс Книги

Польза:

  • Доступ к обширной библиотеке книг, включая аудиокниги и комиксы
  • Возможность загружать собственные файлы в форматах FB2 или EPUB
  • Синхронизация чтения между аудио- и текстовой версией книги

Частота использования: Несколько раз в неделю

Мой Мосэнергосбыт

Польза:

  • Управление несколькими лицевыми счетами в одном приложении
  • Удобная передача показаний счетчиков и оплата без комиссии
  • Push-уведомления о сроках передачи показаний и необходимости оплаты

Частота использования: 1-2 раза в месяц

Неудобства приложений

Telegram

1. Отсутствие фильтрации уведомлений по ключевым словам

При подписке на каналы с 20+ постами в день пользователь получает 100% уведомлений, включая нерелевантный контент. Невозможно настроить получение оповещений только для сообщений, содержащих определенные ключевые слова.

2. Ограничение функционала групповых видеозвонков

В групповых звонках с более чем 15 участниками отсутствуют функции: разделение на комнаты, управление правами говорящих, виртуальное поднятие руки. Это снижает эффективность рабочих встреч по сравнению с специализированными платформами.

3. Отсутствие планировщика отправки сообщений

Невозможно запланировать отправку сообщений на определенное время (с точностью до минуты), что критично для пользователей, работающих в 3+ часовых поясах и вынужденных использовать сторонние приложения для отложенной отправки.

Яндекс Книги

1. Высокий порог входа из-за перегруженного интерфейса

Интерфейс содержит 25+ настроек визуального оформления и 15+ параметров синхронизации. Новому пользователю требуется 15-20 минут для базовой настройки против 5-7 минут в конкурирующих приложениях.

2. Низкое качество синтеза речи в 30% книг

В 3 из 10 аудиокниг наблюдается: некорректное произношение сложных терминов (15% слов), неправильные паузы между предложениями, монотонность голоса. Это снижает комфорт прослушивания.

3. Ограниченные возможности кастомизации библиотеки

Доступно только 3 критерия сортировки (по названию, дате добавления, автору) против 7+ в аналогичных сервисах. Невозможно создавать пользовательские коллекции, тегировать книги или применять сложные фильтры для библиотеки от 100+ книг.

Мой Мосэнергосбыт

1. Длительная процедура восстановления доступа

В 30% случаев время восстановления пароля превышает 10 минут. Процесс включает 5+ шагов верификации, что создает проблемы при срочной необходимости передать показания в последний день месяца.

2. Отсутствие расширенной истории потребления

Доступна история показаний только за 3 месяца (90 дней), тогда как для анализа сезонного потребления и выявления аномалий необходимы данные за 12+ месяцев. Невозможно экспортировать данные в CSV/Excel для самостоятельного анализа.

3. Нестабильность работы в критических сценариях

В 20% случаев приложение аварийно завершает работу на этапе оплаты, что требует полного повторного прохождения всех действий. Критично в последние 3 дня месяца при высокой нагрузке на серверы.

Решение выбранной проблемы

Проблема: Отсутствие фильтрации уведомлений по ключевым словам в Telegram. При подписке на каналы с 20+ постами в день пользователь получает 100% уведомлений, включая нерелевантный контент.

Возможные варианты решения:

  1. Система фильтрации на основе ключевых слов - позволит сократить количество нерелевантных уведомлений на 60-80% за счет точных правил фильтрации
  2. Умный алгоритм приоритизации уведомлений - автоматически определит важность контента с точностью до 85% на основе ML-анализа
  3. Режим "Только важные уведомления" - снизит количество оповещений на 70% с возможностью кастомизации критериев важности

Детальное описание решения №1: Система фильтрации на основе ключевых слов

Пользователь сможет настроить персонализированные правила фильтрации через раздел «Настройки » Уведомления и звуки » Уведомления из чатов » Фильтры».

Таблица примеров правил фильтрации:

Ключевые слова Действие
важно, срочно, новость Показывать уведомления
акция, распродажа Скрывать уведомления
упоминание, ответ Показывать с звуком
убийство Скрывать без звука

Схема процесса настройки:

flowchart TD A[Открыть Настройки] --> B[Уведомления и звуки] B --> C[Уведомления из чатов] C --> D[Фильтры] D --> E[Создать новый фильтр] E --> G[Активировать ключевые слова] G --> H[Ввести ключевые слова] H --> I{Выбрать действие} I --> J[Показывать] I --> K[Скрывать] I --> L[Со звуком] I --> M[Без звука] J --> N[Сохранить фильтр] K --> N L --> N M --> N N --> O[Фильтр активен]

Схема работы фильтрации:

flowchart TD A[Новое сообщение] --> B[Проверить фильтры] B --> E{Содержит ключ. слова?} E -- Да --> F[Применить действие] E -- Нет --> G[Пропустить] F --> M[Действие: показать/скрыть] G --> N[Стандартная обработка]

Интерактивный прототип интерфейса настройки фильтров

Ниже представлен интерактивный прототип экрана создания нового правила фильтрации уведомлений на основе ключевых слов. Вы можете взаимодействовать с элементами интерфейса: нажимать на кнопки.

Интерактивный прототип создан в Figma

Инструкция по взаимодействию с прототипом:

  1. Прокрутите вниз до кнопки «Фильтр» в блоке "Уведомления из чатов".
  2. Нажмите на кнопку «Фильтр» в блоке "Уведомления из чатов".
  3. Нажмите на кнопку «Ключевые слова», чтобы написать их.
  4. Введите ключевые слова. (Нажмите на область, где их вводить - они введутся автоматически)
  5. Используйте переключатели, чтобы выбрать действие для уведомлений.

Преимущества:

  • Повышение релевантности получаемых оповещений
  • Уменьшение отвлекающих факторов и повышение продуктивности
  • Гибкая настройка под индивидуальные потребности

Пожелания по развитию в формате User Story

Для приложения Telegram:

User Story #1: Фильтрация уведомлений

Как активный пользователь каналов, я хочу настраивать фильтры уведомлений по ключевым словам, чтобы получать только релевантные мне оповещения и избегать информационного шума.

User Story #2: Планировщик сообщений

Как пользователь, общающийся с людьми в разных часовых поясах, я хочу иметь возможность планировать отправку сообщений на определенное время, чтобы не беспокоить получателей ночью и обеспечить своевременную доставку важной информации.

User Story #3: Улучшенные групповые видеозвонки

Как организатор групповых встреч, я хочу иметь расширенные функции для видеозвонков (разделение на комнаты, поднятие руки, улучшенный контроль доступа), чтобы проводить более эффективные и организованные онлайн-встречи и вебинары.
